The ADDIE Model

ADDIE is the most widely used framework in instructional design. It provides a systematic, repeatable process for creating effective learning experiences — whether you're building a 5-minute compliance module or a semester-long university course.

Learning Objectives

  • Name and describe each phase of ADDIE
  • Identify key activities within each phase
  • Explain how ADDIE supports iterative design
  • Apply the framework to a real project scenario

Phase 1 of 5

Analysis — Understanding the Problem

Before designing a single slide or writing a single objective, you must understand the problem you're solving. The Analysis phase prevents you from building the wrong solution — even if you build it brilliantly.

🔍 What happens in Analysis?

You investigate the performance gap, understand your audience, clarify constraints, and define what success looks like.

Needs Analysis Audience Profile Task Analysis Gap Analysis Stakeholder Interviews Resource Inventory
  • What performance problem are we solving?
  • Who are the learners — experience, role, prior knowledge?
  • Is training actually the right solution?
  • What are the constraints (budget, time, tech)?
  • How will we measure success?
  • Needs analysis report
  • Learner persona / audience profile
  • Project scope and constraints document
  • Recommended solution (may not be training)
  • Skipping analysis and jumping straight to design
  • Assuming training will fix a motivation problem
  • Not consulting the actual target learners
  • Ignoring technical and accessibility constraints
Phase 2 of 5

Design — Blueprinting the Solution

Design is where analysis findings become a concrete plan. You're not building content yet — you're creating the blueprint that will guide development.

✏️ What happens in Design?

You write learning objectives, sequence content, design assessments, and create storyboards or wireframes before a single piece of content is built.

Learning Objectives Content Outline Storyboarding Assessment Design Style Guide Prototype

💡 Reflection Prompt

Think about a training you've experienced that felt disorganised or irrelevant. What analysis or design step do you think was skipped? How could a structured Design phase have improved it?

Writing Strong Learning Objectives

Use Bloom's Taxonomy action verbs. A strong objective includes: Audience + Behaviour + Condition + Degree (the ABCD model).

Example: "By the end of this module, the learner will be able to identify the five phases of ADDIE and describe one key activity within each phase."

Phases 3 · 4 · 5

Develop · Implement · Evaluate

With analysis and design complete, you move into building, deploying, and measuring your learning solution.

🛠️ Develop

Build the actual content — slides, videos, interactions, assessments. Use your storyboard as a guide. Conduct pilot testing and gather SME review before launch.

Content Creation Authoring Tool Build Media Production Alpha / Beta Review

🚀 Implement

Deploy to your LMS or delivery platform. Train facilitators if needed. Enrol learners. Monitor for technical issues and early completion data.

LMS Upload Facilitator Training Learner Onboarding Technical Testing

📊 Evaluate

Use Kirkpatrick's Four Levels — Reaction, Learning, Behaviour, Results — to measure training effectiveness and identify improvements.

Level 1: Reaction Level 2: Learning Level 3: Behaviour Level 4: Results
